GMRES -- Generalized Minimal Residual Method

Problem Statement

Solve \(Ax = b\) where \(A \in \mathbb{R}^{n \times n}\) is non-singular but possibly non-symmetric. CG requires symmetry and positive definiteness; GMRES works for any invertible \(A\).

GMRES finds the iterate \(x_k \in x_0 + \mathcal{K}_k(A, r_0)\) that minimizes the 2-norm of the residual over the Krylov subspace:

\[x_k = \operatorname*{argmin}_{x \in x_0 + \mathcal{K}_k} \|Ax - b\|_2, \qquad \mathcal{K}_k = \operatorname{span}\!\bigl\{r_0, Ar_0, \ldots, A^{k-1}r_0\bigr\}\]

The basis grows by one vector per iteration. **Restarted GMRES( \(m\))** caps memory at \(m\) vectors by restarting from the current iterate every \(m\) steps.

Arnoldi Process

The Krylov basis is built via the Arnoldi recurrence using Modified Gram-Schmidt:

function Arnoldi(A, v_j, V, H, j):
w <- A v_j
// Project out components along all previous basis vectors:
// \f$h_{ij} = v_i^T w,\quad w \leftarrow w - h_{ij}\, v_i\f$ (Modified Gram-Schmidt)
for i = 0 to j:
H[j,i] <- w^T V[i]
w <- w - H[j,i] * V[i]
H[j, j+1] <- ||w||_2
v_{j+1} <- w / H[j, j+1] // next basis vector

After \(k\) steps this produces the Arnoldi relation:

\[A V_k = V_{k+1} \bar{H}_k\]

where \(V_k = [v_1 \mid \cdots \mid v_k]\) has orthonormal columns and \(\bar{H}_k \in \mathbb{R}^{(k+1) \times k}\) is the upper Hessenberg matrix of Gram-Schmidt coefficients.

The GMRES Least-Squares Problem

The residual at step \(k\) satisfies:

\[\|Ax_k - b\|_2 = \|A(x_0 + V_k y_k) - b\|_2 = \|AV_k y_k - r_0\|_2\]

Using the Arnoldi relation and the fact that \(V_{k+1}\) is orthonormal:

\[\|Ax_k - b\|_2 = \|V_{k+1}\bar{H}_k y_k - \beta e_1\|_2 = \|\bar{H}_k y_k - \beta e_1\|_2, \quad \beta = \|r_0\|_2\]

This is a small \((k+1) \times k\) least-squares problem – solved via Givens rotations applied incrementally (one new rotation per step):

\[\begin{pmatrix} c_j & s_j \\ -s_j & c_j \end{pmatrix} \begin{pmatrix} H_{jj} \\ H_{j,j+1} \end{pmatrix} = \begin{pmatrix} r \\ 0 \end{pmatrix}, \qquad c_j = \frac{H_{jj}}{d},\ s_j = \frac{H_{j,j+1}}{d},\ d = \sqrt{H_{jj}^2 + H_{j,j+1}^2}\]

The rotated RHS vector \(g\) gives: \(|g_{j+1}| = \|r_{j+1}\|_2\) – monitor this for convergence without computing \(x_k\) explicitly.

Algorithm: Restarted GMRES(\f$m\f$)

function GMRES(A, b, x, tol, max_iter, restart):
while total_iters < max_iter:
r <- b - Ax; beta <- ||r||_2
if beta < tol: return (x, converged)
V[0] <- r / beta
g[0] <- beta; g[1:m+1] <- 0
for j = 0 to restart-1:
// Arnoldi step (MGS)
w <- A V[j]
for i = 0 to j:
H[j,i] <- w^T V[i]; w <- w - H[j,i] * V[i]
H[j,j+1] <- ||w||; if H[j,j+1] < eps: break
V[j+1] <- w / H[j,j+1]
// Apply previous Givens rotations to column j of H
for i = 0 to j-1:
[H[j,i], H[j,i+1]] <- G_i [H[j,i], H[j,i+1]]
// New Givens rotation to zero H[j,j+1]
d <- sqrt(H[j,j]^2 + H[j,j+1]^2)
c[j] <- H[j,j]/d; s[j] <- H[j,j+1]/d
H[j,j] <- c[j]*H[j,j] + s[j]*H[j,j+1]; H[j,j+1] <- 0
g[j+1] <- -s[j]*g[j]; g[j] <- c[j]*g[j]
if |g[j+1]| < tol: goto back_solve
back_solve:
m_actual <- j+1
// Back-solve upper triangular H[0:m,0:m] y = g[0:m]
y[m-1] <- g[m-1] / H[m-1,m-1]
for i = m-2 downto 0:
y[i] <- (g[i] - sum_{k=i+1}^{m-1} H[k,i]*y[k]) / H[i,i]
x <- x + sum_{i=0}^{m-1} y[i] * V[i]
if converged: break

Memory: \((m+1)\) vectors of length \(n\) -> \(O(mn)\) storage. Typical restart: \(m \in [30, 300]\).

Flops per restart: \(2mn^2\) (matvecs) \(+ 2m^2n\) (MGS) \(+ O(m^2)\) (Givens). For \(m \ll n\), the matvec dominates.

Convergence

For \(A = I + E\) with \(\|E\|\) small, GMRES converges like CG in \(O(\sqrt{\kappa})\) iterations. In general, GMRES convergence is governed by the polynomial approximation problem:

\[\frac{\|r_k\|_2}{\|r_0\|_2} \leq \min_{\substack{p_k \in \mathcal{P}_k \\ p_k(0) = 1}} \max_{\lambda \in \sigma(A)} |p_k(\lambda)|\]

  • **Normal \(A\)** (i.e., \(A = U\Lambda U^H\)): reduces to approximation of zero on the spectrum, similar to CG
  • Clustered spectrum: fast convergence – a low-degree polynomial can be small on a few eigenvalue clusters
  • Stagnation: GMRES can plateau many steps then converge suddenly (e.g., one outlier eigenvalue far from the main cluster)

Preconditioning

Left preconditioning: solve \(M^{-1}Ax = M^{-1}b\):

\[\text{GMRES on } M^{-1}A \text{ minimizes } \|M^{-1}r_k\|_2\]

Right preconditioning: solve \(AM^{-1}(Mx) = b\), then \(x \leftarrow M^{-1}(Mx)\):

\[\text{GMRES on } AM^{-1} \text{ minimizes } \|r_k\|_2 \quad \text{(true residual)}\]

Right preconditioning is usually preferred because it minimizes the physically meaningful residual norm. Left preconditioning minimizes \(\|M^{-1}r\|_2\) which depends on \(M\).

Common preconditioners for non-symmetric problems:

  • **ILU( \(k\))**: incomplete LU with level- \(k\) fill – general, moderate improvement
  • Block Jacobi: partition rows into blocks, invert each block exactly – parallelizes
  • SSOR: symmetric successive over-relaxation – cheap, effective for M-matrices
  • AMG: algebraic multigrid – near \(O(n)\) convergence for elliptic PDE

Variants

Variant Memory Notes
Full GMRES \(O(n^2)\) No restart; \(n\)-step convergence; impractical for large \(n\)
GMRES( \(m\)) \(O(mn)\) Restart every \(m\) steps; our implementation
FGMRES \(2O(mn)\) Flexible preconditioner (can change each step)
LGMRES \(O(mn)\) Augment Krylov space with error vectors from previous restarts
BiCGSTAB \(O(n)\) Short recurrence; non-symmetric; not min-residual but cheaper/restart-free
MINRES \(O(n)\) Symmetric indefinite \(A\); short recurrence; exact residual minimization

Performance Optimization

BLAS-2 Arnoldi

Standard MGS is a sequence of BLAS-1 DOT + AXPY at step \(j\): \(O(j \cdot n)\) cost, \(O(m^2 n)\) total. Replace with BLAS-2 DGEMV:

// All j+1 dot products at once:
h = dgemv(V[:,0:j+1]^T, w) // (j+1) x n times n x 1 -> O(jn), one BLAS call
w -= dgemv(V[:,0:j+1], h) // n x (j+1) times (j+1) x 1

Replaces \(j\) DDOT \(+ j\) DAXPY calls with 2 DGEMV calls, improving cache efficiency ( \(V[:,0:j+1]\) loaded once for both operations).

Block Arnoldi (BLAS-3)

Orthogonalize \(b\) vectors simultaneously:

\[[v_{j+1}, \ldots, v_{j+b}] = AV[:,j:j+b] \quad \to \quad \text{DGEMM}\]

\[H \leftarrow V[:,0:j]^T W, \quad W \leftarrow W - V[:,0:j] H \quad \to \quad \text{2 DGEMM calls}\]

This converts BLAS-1/2 operations to BLAS-3 DGEMM, dramatically improving arithmetic intensity on cache-blocked hardware.

Communication-Avoiding GMRES (CA-GMRES)

Standard GMRES performs one global all-reduce per iteration for \(\|w\|\) in MGS. For \(p\) distributed processes, this is \(O(\log p)\) latency per step.

CA-GMRES computes \(s\) basis vectors per communication round using matrix powers:

\[[v,\, Av,\, A^2v,\, \ldots,\, A^s v]\]

(a Newton or Chebyshev basis), then orthogonalizes the block. This reduces all-reduces by factor \(s\) at the cost of potential loss of numerical orthogonality (mitigated by careful basis selection).

SIMD for Dense Matvec and MGS

Matvec (row \(i\) dot with \(p\)):

__m256d acc = _mm256_setzero_pd();
for (idx j = 0; j < n; j += 4)
acc = _mm256_fmadd_pd(_mm256_load_pd(&A(i,j)),
_mm256_load_pd(&p[j]), acc);
w[i] = hsum(acc);

Fused dot + AXPY for MGS (two passes required since \(h\) is needed for the update):

// Pass 1: h = w^T V[i]
__m256d vdot = _mm256_setzero_pd();
for (idx k = 0; k < n; k += 4)
vdot = _mm256_fmadd_pd(_mm256_load_pd(&w[k]),
_mm256_load_pd(&V[i][k]), vdot);
real h = hsum(vdot);
// Pass 2: w -= h * V[i]
__m256d vh = _mm256_broadcast_sd(&h);
for (idx k = 0; k < n; k += 4) {
__m256d vw = _mm256_load_pd(&w[k]);
__m256d vv = _mm256_load_pd(&V[i][k]);
_mm256_store_pd(&w[k], _mm256_fnmadd_pd(vh, vv, vw));
}
