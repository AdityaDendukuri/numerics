One-Sided Jacobi SVD

Problem Statement

Given \(A \in \mathbb{R}^{m \times n}\) with \(m \geq n\), find the singular value decomposition:

\[A = U \Sigma V^T, \qquad U \in \mathbb{R}^{m \times r},\ \Sigma = \operatorname{diag}(\sigma_1, \ldots, \sigma_r),\ V \in \mathbb{R}^{n \times r}\]

where \(r = \min(m, n)\), \(U^TU = I\), \(V^TV = I\), and \(\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_r \geq 0\) are the singular values.

The one-sided Jacobi method applies Givens rotations to the columns of \(A\) from the right until they are mutually orthogonal. Column norms then give singular values; normalized columns give left singular vectors.

Mathematical Foundation

Relation to the Eigenvalue Problem

The singular values of \(A\) are the square roots of the eigenvalues of \(A^TA\) (or \(AA^T\)):

\[A^TA = V \Sigma^2 V^T\]

Zeroing \([A^TA]_{pq}\) is equivalent to orthogonalizing columns \(p\) and \(q\) of \(A\). The one-sided method works directly on \(A\), never forming \(A^TA\) explicitly (which would square the condition number).

The Column Orthogonality Condition

Define the inner products of columns \(p\) and \(q\):

\[\alpha = \|a_p\|^2 = \sum_i A_{ip}^2, \qquad \beta = \|a_q\|^2 = \sum_i A_{iq}^2, \qquad \gamma = a_p \cdot a_q = \sum_i A_{ip} A_{iq}\]

The relative cosine \(|\gamma|/\sqrt{\alpha\beta} = |\cos\angle(a_p, a_q)|\) measures deviation from orthogonality. Rotation is skipped when \(|\gamma|/\sqrt{\alpha\beta} < \varepsilon_{\text{tol}}\).

Rotation Angle to Zero \f$[A^TA]_{pq}\f$

The right rotation \(A \leftarrow AG\) with \(G\) a Givens matrix in the \((p,q)\) plane zeros \([A^TA]_{pq} = \gamma\) when (setting the new inner product to zero):

\[\tan(2\theta) = \frac{2\gamma}{\beta - \alpha} \qquad \Longrightarrow \qquad \zeta = \frac{\beta - \alpha}{2\gamma}, \quad t = \frac{\operatorname{sign}(\zeta)}{|\zeta| + \sqrt{1 + \zeta^2}}\]

\[c = \frac{1}{\sqrt{1+t^2}}, \qquad s = ct\]

This formula uses the smaller root of \(t^2 + 2\zeta t - 1 = 0\) – consistent with \(|t| \leq 1\), i.e., \(|\theta| \leq \pi/4\).

Sign verification: with rotation \(a_p' = c\, a_p - s\, a_q\), \(a_q' = s\, a_p + c\, a_q\), the new inner product is:

\[a_p'^T a_q' = cs(\alpha - \beta) + (c^2 - s^2)\gamma = 0 \quad \checkmark\]

(this is \(c^2 - s^2 = 1 - 2t^2/(1+t^2)\) and \(2cs = 2t/(1+t^2)\), so \((c^2-s^2)\gamma + 2cs \cdot (\alpha-\beta)/2 = 0\) by construction of \(t\)).

Algorithm

function svd(A, tol, max_sweeps):
m, n <- A.rows, A.cols
r <- min(m, n)
V <- I_n // accumulates right singular vectors
for sweep = 1 to max_sweeps:
max_cos <- 0
for p = 0 to r-2:
for q = p+1 to r-1:
// Inner products of columns p and q
alpha <- sum_i A[i,p]^2
beta <- sum_i A[i,q]^2
gamma <- sum_i A[i,p] * A[i,q]
if alpha < 1e-300 or beta < 1e-300: continue
cos_pq <- |gamma| / sqrt(alpha * beta)
max_cos <- max(max_cos, cos_pq)
if cos_pq < tol: continue // already orthogonal enough
// Rotation parameters
zeta <- (beta - alpha) / (2 * gamma)
t <- sign(zeta) / (|zeta| + sqrt(1 + zeta^2))
c <- 1 / sqrt(1 + t^2); s <- c * t
// Update columns of A
for i = 0..m-1:
(A[i,p], A[i,q]) <- (c*A[i,p] - s*A[i,q], s*A[i,p] + c*A[i,q])
// Accumulate right singular vectors
for i = 0..n-1:
(V[i,p], V[i,q]) <- (c*V[i,p] - s*V[i,q], s*V[i,p] + c*V[i,q])
if max_cos < tol: break // all column pairs orthogonal enough
// Extract singular values and left singular vectors
for j = 0..r-1:
sigma[j] <- ||A[:,j]||
U[:,j] <- A[:,j] / sigma[j] (if sigma[j] > 0)
// Sort descending by singular value; permute U, V columns to match
selection_sort(sigma, U, V)
// Return economy Vt (r x n)
Vt[i,j] <- V[j,i] for i < r
return (U, sigma, Vt, sweeps, converged)

Complexity: \(O(r^2)\) rotations per sweep \(\times\) \(O(m + n)\) flops per rotation = \(O(r^2(m+n))\) per sweep.

Convergence: typically 5-10 sweeps for double precision, similar to the symmetric Jacobi eigensolver.

Convergence Analysis

Define the off-orthogonality of \(A\):

\[\psi(A) = \sum_{p < q} \bigl([A^TA]_{pq}\bigr)^2 = \sum_{p < q} \gamma_{pq}^2\]

Each rotation decreases \(\psi\) by \(\gamma_{pq}^2\):

\[\psi(A') = \psi(A) - \gamma_{pq}^2\]

After a full sweep visiting all \(\binom{r}{2}\) pairs:

\[\psi(A_{\text{sweep}+1}) \leq \left(1 - \frac{2}{r(r-1)}\right) \psi(A_{\text{sweep}}) \qquad \text{(linear phase)}\]

Ultimate quadratic convergence: once \(\max_{p \neq q} |[A^TA]_{pq}|\) is small, convergence transitions to a quadratic phase – same analysis as symmetric Jacobi applied to \(A^TA\).

Accuracy Properties

One-sided Jacobi SVD is among the most accurate SVD algorithms:

  1. High relative accuracy for small singular values: unlike bidiagonalization + QR, the Jacobi SVD can compute small singular values to high relative accuracy even when \(\kappa(A) \gg 1/\varepsilon_{\text{mach}}\), provided the columns of \(A\) are well-scaled. Formally, if \(\sigma_k / \sigma_1 = O(\varepsilon_{\text{mach}})\), Jacobi still computes \(\sigma_k\) with relative error \(O(\varepsilon_{\text{mach}})\).
  2. Column-scaled accuracy: the relative accuracy of \(\sigma_k\) is \(O(\varepsilon_{\text{mach}} \cdot \kappa(D^{-1}A))\) where \(D = \operatorname{diag}(\|a_1\|, \ldots, \|a_n\|)\) is the column scaling. If \(A\) is column-equilibrated ( \(D \approx I\)), all singular values are computed to full relative precision.

This contrasts with standard bidiagonalization (LAPACK dgesdd), which computes all singular values to absolute accuracy \(O(\varepsilon_{\text{mach}} \|A\|)\) – relative accuracy only for \(\sigma_k \gtrsim \varepsilon_{\text{mach}} \|A\|\).

For \(m \gg n\) (tall-skinny): first compute thin QR ( \(A = \hat{Q}\hat{R}\)), then Jacobi SVD on \(\hat{R} \in \mathbb{R}^{n \times n}\). Left singular vectors: \(U = \hat{Q}\hat{U}\).

Performance Optimization

Current Implementation

Two nested loops over column pairs \((p, q)\), each requiring:

  • Three inner products ( \(\alpha\), \(\beta\), \(\gamma\)): \(3m\) FMAs -> bandwidth limited
  • Column rotation of \(A\): \(4m\) FMAs
  • Column rotation of \(V\): \(4n\) FMAs

Total per rotation: \((3 + 4)m + 4n \approx 7m\) flops for \(m \gg n\).

SIMD for Column Inner Products and Rotation

Fuse the three inner product computations and the rotation in one pass:

// Step 1: compute alpha, beta, gamma (three accumulators, one pass over columns p,q)
__m256d va = _mm256_setzero_pd(); // alpha
__m256d vb = _mm256_setzero_pd(); // beta
__m256d vg = _mm256_setzero_pd(); // gamma
for (idx i = 0; i < m; i += 4) {
__m256d ap = _mm256_load_pd(&A(i, p));
__m256d aq = _mm256_load_pd(&A(i, q));
va = _mm256_fmadd_pd(ap, ap, va);
vb = _mm256_fmadd_pd(aq, aq, vb);
vg = _mm256_fmadd_pd(ap, aq, vg);
}
alpha = hsum(va); beta = hsum(vb); gamma = hsum(vg);
// Step 2: rotation (second pass)
__m256d vc = _mm256_broadcast_sd(&c);
__m256d vs = _mm256_broadcast_sd(&s);
for (idx i = 0; i < m; i += 4) {
__m256d ap = _mm256_load_pd(&A(i, p));
__m256d aq = _mm256_load_pd(&A(i, q));
_mm256_store_pd(&A(i,p), _mm256_fmsub_pd(vc,ap, _mm256_mul_pd(vs,aq)));
_mm256_store_pd(&A(i,q), _mm256_fmadd_pd(vs,ap, _mm256_mul_pd(vc,aq)));
}

Two passes per rotation pair (one for inner products, one for the update). The data \(A[:,p]\) and \(A[:,q]\) are loaded twice; for column-major storage they are contiguous and fit in L1 for small \(m\).

Fused Single-Pass Update

When \(m \leq L_1 / (2 \times 8\,\text{bytes})\) (both columns fit in L1, i.e., \(m \leq 4096\) for 64 KB L1):

// Read-compute-write in one pass: inner products AND rotation together
// Requires updating V with the same (c,s) computed on-the-fly -- NOT POSSIBLE
// (c,s depend on gamma which requires a full pass first)

Cannot fuse into one pass because \(c, s\) depend on \(\gamma\) which requires a full scan. For large \(m\) (columns don't fit in cache), the two-pass approach causes 2 column pair reads from DRAM – unavoidable unless the rotation is batched.

Batched Rotations (Row-Blocked Update)

Process all \(\binom{r}{2}\) rotations in a sweep as a sequence of BLAS-2 DROT operations, then bulk-apply them to \(V\) using DGEMM:

// Phase 1: Compute all (zeta, t, c, s) for the sweep
for all (p,q) pairs: compute (c_{pq}, s_{pq}) from A
// Phase 2: Apply rotations to A column pairs (BLAS-1 DROT x r^2/2)
for all (p,q) pairs: drot(m, A[:,p], A[:,q], c_pq, s_pq)
// Phase 3: Accumulate V as product of all Givens matrices in DGEMM form
// V_new = V_old * G_1 * G_2 * ... (product of Givens -> QR of random matrix -> compact)

For the \(V\) accumulation, represent the sequence of \(\binom{r}{2}\) Givens rotations as a single orthogonal matrix via their WY product representation, then apply with one DGEMM of size \(n \times n \times n\) – BLAS-3 instead of \(O(r^2)\) BLAS-1 DROT calls.

Pre-Processing: Column Equilibration

Before running Jacobi, normalize columns:

\[A \leftarrow A D^{-1}, \quad D = \operatorname{diag}(\|a_1\|, \ldots, \|a_n\|)\]

After convergence, \(\sigma_j^{\text{actual}} = \sigma_j^{\text{computed}} \times D_{jj}\).

Column equilibration improves convergence rate (all column pairs start at roughly equal norm -> more uniform convergence) and is essential for the high relative accuracy property.

Block One-Sided Jacobi

Instead of scalar rotations, process \(b \times b\) blocks of column pairs simultaneously:

  1. Extract the \(2b \times 2b\) block \(B = \begin{pmatrix} A[:,P]^T A[:,P] & A[:,P]^T A[:,Q] \\ A[:,Q]^T A[:,P] & A[:,Q]^T A[:,Q] \end{pmatrix}\)
  2. Compute orthogonal \(G \in \mathbb{R}^{2b \times 2b}\) that diagonalizes \(B\) via small \(2b \times 2b\) Jacobi
  3. Update \([A[:,P], A[:,Q]] \leftarrow [A[:,P], A[:,Q]] \cdot G\) via DGEMM

The dominant cost is the DGEMM: \(O(m \cdot 2b \cdot 2b) = O(4mb^2)\) with arithmetic intensity \(O(b)\). This is the block Jacobi SVD – achieves near-peak GFLOP/s for the column update phase.
