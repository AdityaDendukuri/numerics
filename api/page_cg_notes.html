Conjugate Gradient and the Thomas Algorithm

Part I – Conjugate Gradient (CG)

Problem Statement

Solve \(Ax = b\) where \(A \in \mathbb{R}^{n \times n}\) is symmetric positive definite (SPD). CG minimizes the \(A\)-norm of the error over the \(k\)-dimensional Krylov subspace:

\[\mathcal{K}_k(A, r_0) = \operatorname{span}\!\bigl\{r_0,\, Ar_0,\, A^2r_0,\, \ldots,\, A^{k-1}r_0\bigr\}\]

The iterate \(x_k\) is the unique point in \(x_0 + \mathcal{K}_k\) minimizing \(\|x - x^*\|_A = \sqrt{(x-x^*)^T A (x-x^*)}\).

The search direction \(p_k\) is made ** \(A\)-conjugate** to all previous directions ( \(p_k^T A p_j = 0\) for \(j < k\)), so each step minimizes the error along \(p_k\) without disturbing prior progress.

Algorithm

function CG(A, b, x_0, tol, max_iter):
r <- b - A x_0
p <- r
rho_old <- r^T r
for iter = 1, 2, ..., max_iter:
w <- A p // matvec: O(n^2) dense, O(nnz) sparse
alpha <- rho_old / (p^T w) // step length
x <- x + alpha * p
r <- r - alpha * w
rho_new <- r^T r
if sqrt(rho_new) < tol: return x, converged
beta <- rho_new / rho_old // Gram-Schmidt coefficient
p <- r + beta * p // new A-conjugate direction
rho_old <- rho_new

Flops per iteration: \(2n^2\) (matvec) \(+ 6n\) (dot, axpy, scale) \(\approx 2n^2\) for large \(n\).

Memory: vectors \(r, p, w\) plus the matrix – \(O(n^2)\) for dense \(A\), or \(O(\text{nnz})\) for sparse. CG never modifies \(A\).

Convergence

CG terminates in at most \(n\) iterations in exact arithmetic. In floating-point, convergence requires \(O(\sqrt{\kappa})\) iterations where \(\kappa = \kappa_2(A) = \lambda_{\max}/\lambda_{\min}\):

\[\frac{\|e_k\|_A}{\|e_0\|_A} \leq 2\left(\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1}\right)^k\]

For poorly conditioned \(A\) ( \(\kappa \gg 1\)), CG stagnates without preconditioning.

Eigenvalue clustering accelerates convergence: if all eigenvalues except one small cluster lie in a single interval, CG quickly annihilates the polynomial responsible for those eigenvalues and then reduces to a problem of size equal to the cluster count.

Preconditioned CG (PCG)

Replace \(A\) with \(M^{-1}A\) where \(M \approx A\) is cheap to invert. The modified iteration solves \(M^{-1/2} A M^{-1/2} \tilde{y} = M^{-1/2}b\), which has condition number \(\kappa(M^{-1}A)\).

function PCG(A, M_inv, b, x, tol):
r <- b - Ax
z <- M_inv(r) // preconditioner solve
p <- z
rho_old <- r^T z // note: r^T z, not r^T r
for iter = ...:
w <- Ap
alpha <- rho_old / (p^T w)
x <- x + alpha*p; r <- r - alpha*w
z <- M_inv(r)
rho_new <- r^T z
beta <- rho_new / rho_old
p <- z + beta*p
rho_old <- rho_new

Common preconditioners:

  • Diagonal (Jacobi): \(M = \operatorname{diag}(A)\) – always cheap, effective when \(A\) is diagonally dominant
  • Incomplete Cholesky (IC0): \(M = \hat{L}\hat{L}^T\) where \(\hat{L}\) has the same sparsity as \(\operatorname{tril}(A)\)
  • Algebraic multigrid (AMG): \(\kappa(M^{-1}A) \approx O(1)\) for Poisson-type problems

Matrix-Free CG

cg_matfree accepts a MatVecFn callback instead of an explicit matrix, enabling:

  • Implicit time stepping ( \(A = M + \Delta t\, K\), never assembled)
  • Spectral methods ( \(A\) applied via FFT in \(O(n \log n)\))
  • Structured operators (Toeplitz, circulant – \(O(n)\) storage)

Part II – Thomas Algorithm (Tridiagonal Solver)

Problem Statement

Solve the tridiagonal system \(Tx = d\) where

\[T = \begin{pmatrix} b_0 & c_0 & & \\ a_1 & b_1 & c_1 & \\ & a_2 & b_2 & \ddots \\ & & \ddots & \ddots & c_{n-2} \\ & & & a_{n-1} & b_{n-1} \end{pmatrix}\]

This is LU factorization specialized to tridiagonal structure: \(O(n)\) work instead of \(O(n^2)\) for dense forward elimination.

Algorithm

function Thomas(a, b, c, d):
// a: sub-diagonal (length n-1)
// b: main diagonal (length n, modified in-place)
// c: super-diagonal (length n-1)
// d: RHS (length n, modified in-place)
// Forward sweep: eliminate sub-diagonal
// \f$w_i = a_i / b_{i-1},\quad b_i \leftarrow b_i - w_i c_{i-1},\quad d_i \leftarrow d_i - w_i d_{i-1}\f$
for i = 1 to n-1:
w <- a[i] / b[i-1]
b[i] <- b[i] - w * c[i-1]
d[i] <- d[i] - w * d[i-1]
// Backward substitution: \f$x_i = \bigl(d_i - c_i\, x_{i+1}\bigr) / b_i\f$
x[n-1] <- d[n-1] / b[n-1]
for i = n-2 downto 0:
x[i] <- (d[i] - c[i] * x[i+1]) / b[i]
return x

Complexity: \(8n\) flops.

Stability: unconditionally stable when \(A\) is SPD or strictly diagonally dominant ( \(|b_i| > |a_i| + |c_i|\) for all \(i\)).

Applicability

Tridiagonal systems arise everywhere in numerical PDE:

  • 1D finite difference / finite element discretization
  • Implicit time integration (Crank-Nicolson for the heat equation \(u_t = u_{xx}\))
  • Cubic spline interpolation (natural and clamped boundary conditions)
  • ADI (alternating direction implicit) splitting for 2D/3D parabolic equations

Performance Optimization

CG: SIMD for Dense Matvec

The dominant cost in each CG iteration is the matvec \(w \leftarrow Ap\). For row-major dense \(A\):

for (idx i = 0; i < n; ++i) {
__m256d acc = _mm256_setzero_pd();
for (idx j = 0; j < n; j += 4)
acc = _mm256_fmadd_pd(_mm256_load_pd(&A(i,j)),
_mm256_load_pd(&p[j]), acc);
w[i] = hsum(acc);
}

FMA throughput on AVX2: \(2 \times 4 = 8\) flops/cycle. For large \(n\), performance is memory-bandwidth limited.

CG: Pipelining (Chronopoulos-Gear)

Standard CG requires two global synchronizations per iteration (one for \(\rho = r^Tr\) and one for \(\sigma = p^TAp\)). The Chronopoulos-Gear variant (s-step CG) computes \(r^Tr\) and \(p^TAp\) in the same matvec loop:

\[\sigma \leftarrow \sum_i w_i p_i, \quad \rho \leftarrow \sum_i r_i^2 \qquad \text{computed simultaneously in one pass over } w, p, r\]

This halves memory bandwidth and reduces synchronization – critical for distributed-memory CG.

Thomas: SIMD over Multiple RHS

Thomas is memory-bandwidth bound: each element is loaded and stored once. For \(m\) independent RHS, unroll over RHS with SIMD – process 4 or 8 simultaneously:

// Forward sweep: process 4 RHS at once
for (idx i = 1; i < n; ++i) {
real w = a[i] / b_work[i-1];
b_work[i] -= w * c[i-1]; // scalar (shared b)
// Vectorised over 4 RHS:
__m256d vw = _mm256_broadcast_sd(&w);
__m256d di = _mm256_load_pd(&D(i, 0));
__m256d dim1 = _mm256_load_pd(&D(i-1, 0));
_mm256_store_pd(&D(i, 0), _mm256_fnmadd_pd(vw, dim1, di));
}
