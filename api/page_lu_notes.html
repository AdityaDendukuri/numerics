LU Factorization with Partial Pivoting

Overview

Factors a non-singular \(A \in \mathbb{R}^{n \times n}\) as

\[PA = LU\]

where \(P\) is a permutation, \(L\) is unit lower triangular, and \(U\) is upper triangular. Partial pivoting selects the pivot row to maximize \(|U_{kk}|\) at each step, keeping \(|L_{ij}| \leq 1\) for all \(i > j\) and preventing catastrophic cancellation.

Algorithm

Doolittle LU with partial pivoting stores \(L\) (below diagonal) and \(U\) (diagonal and above) in the same working matrix \(M\). The diagonal of \(L\) is implicitly 1.

function LU(A):
n <- A.rows
M <- copy of A
piv[0..n-1] <- identity permutation
for k = 0 to n-1:
// Partial pivot: find row with max |M[i,k]| for i >= k
pivot_row <- argmax_{i >= k} |M[i,k]|
piv[k] <- pivot_row
swap rows k and pivot_row of M
if |M[k,k]| < eps: mark singular; continue
// Column k of L: multipliers stored below diagonal
for i = k+1 to n-1:
M[i,k] <- M[i,k] / M[k,k]
// Schur complement (rank-1 update of trailing submatrix)
for i = k+1 to n-1:
lik <- M[i,k]
for j = k+1 to n-1:
M[i,j] <- M[i,j] - lik * M[k,j]
return (M, piv)

Complexity: \(\frac{2}{3}n^3\) flops for factorization.

Forward/Backward Substitution

Given \(PAx = b\), i.e., \(LUx = Pb\):

Step 1 – apply permutation: \(y \leftarrow Pb\).

Step 2 – forward substitution \(Ly = Pb\) (unit diagonal):

\[y_i = y_i - \sum_{j=0}^{i-1} L_{ij}\, y_j, \quad i = 1, \ldots, n-1\]

Step 3 – backward substitution \(Ux = y\):

\[x_i = \frac{1}{U_{ii}}\!\left(y_i - \sum_{j=i+1}^{n-1} U_{ij}\, x_j\right), \quad i = n-1, \ldots, 0\]

function LU_solve(M, piv, b):
y <- copy of b
for k = 0..n-1: swap y[k], y[piv[k]] // permutation
for i = 1..n-1: // forward
for j = 0..i-1: y[i] -= M[i,j] * y[j]
for i = n-1 downto 0: // backward
for j = i+1..n-1: y[i] -= M[i,j] * y[j]
y[i] /= M[i,i]
return y

Complexity: \(n^2\) flops for each sweep.

Determinant and Inverse

\[\det(A) = (-1)^{\text{swaps}} \prod_{i=0}^{n-1} U_{ii}\]

where \(\text{swaps}\) = number of non-trivial pivots ( \(\text{piv}[k] \neq k\)).

Inverse: solve \(AX = I\) column by column – \(n\) calls to LU_solve, each \(O(n^2)\), total \(O(n^3)\). Never compute \(A^{-1}\) explicitly if only \(A^{-1}b\) is needed; just solve \(Ax = b\) directly.

Stability

LU with partial pivoting is backward stable. The growth factor \(\rho(n)\) is bounded at \(2^{n-1}\) in theory but observed as \(O(n^{1/2})\) in practice. Complete pivoting (swapping rows and columns) has tighter theoretical bounds but is rarely needed; partial pivoting suffices for virtually all practical matrices.

Performance Optimization

Current Implementation

Naive in-place Doolittle. The innermost Schur complement update

\[M_{ij} \leftarrow M_{ij} - \ell_{ik}\, U_{kj}, \quad i,j > k\]

is a rank-1 SAXPY over row \(j\). With row-major storage, access to \(M[k,j]\) (a row) is contiguous, but the multiplier column \(M[\cdot,k]\) is strided – one cache miss per row for large \(n\).

Level-3 BLAS: Blocked LU (LAPACK <tt>dgetrf</tt>)

Partition \(A\) into panels of width \(n_b\) columns (typically \(n_b \in [64, 256]\)):

\[\begin{pmatrix} A_{11} & A_{12} \\ A_{21} & A_{22} \end{pmatrix} \xrightarrow{\text{factor}} \begin{pmatrix} L_{11} U_{11} & U_{12} \\ A_{21} & A_{22} - L_{21}U_{12} \end{pmatrix}\]

for k = 0 to n step nb:
panel_lu(A[k:n, k:k+nb]) // unblocked, O(n * nb^2)
apply_pivots(A[k:n, k+nb:n])
dtrsm(L[k:k+nb, k:k+nb], A[k:k+nb, k+nb:n]) // triangular solve
dgemm(A[k+nb:n, k:k+nb], // Schur complement: BLAS-3!
A[k:k+nb, k+nb:n],
A[k+nb:n, k+nb:n])

The Schur complement update is a dgemm with arithmetic intensity \(O(n_b)\) – reaches near-peak FLOP/s. The panel factorization is \(O(n \cdot n_b^2)\) – cheap relative to the trailing DGEMM.

Block size selection: \(n_b\) should satisfy \(2 n_b \times n_b \times 8\,\text{bytes} \lesssim L_2\) cache. For a 256 KB L2: \(n_b \approx \sqrt{256\text{KB}/16} \approx 128\).

SIMD for the Schur Complement

The rank-1 update inner loop is a DAXPY with FMA:

for (idx j = k+1; j < n; j += 4) {
__m256d vlik = _mm256_broadcast_sd(&lik);
__m256d vukj = _mm256_loadu_pd(&M[k][j]);
__m256d vmij = _mm256_loadu_pd(&M[i][j]);
vmij = _mm256_fnmadd_pd(vlik, vukj, vmij); // M[i,j] -= lik * M[k,j]
_mm256_storeu_pd(&M[i][j], vmij);
}

FMA throughput on AVX2: \(2 \times 4 = 8\) flops/cycle. Align rows to 32-byte boundaries and use _mm256_load_pd instead of _mm256_loadu_pd for maximum throughput.

Recursive LU

Use divide-and-conquer at split \(n/2\) rather than a fixed panel width:

LU_recursive(A[0:n, 0:n]):
LU_recursive(A[0:n/2, 0:n/2]) // left half
dtrsm + dgemm for right block
LU_recursive(A[n/2:n, n/2:n]) // Schur complement

This achieves optimal cache reuse at all levels of the hierarchy without needing to tune \(n_b\) – recursion bottoms out when the block fits in L1. Used in PLASMA and other communication-avoiding libraries.

Multiple Right-Hand Sides

For \(m\) RHS, batch the triangular solves:

\[L\,Y = PB, \quad U\,X = Y\]

Each is a single DTRSM(L, Y) and DTRSM(U, Y) call – triangular solve for a matrix, achieving BLAS-3 performance instead of \(m\) separate DTRSV calls.
