Randomized Truncated SVD

Problem Statement

Given \(A \in \mathbb{R}^{m \times n}\) and a target rank \(k \ll \min(m, n)\), find an approximate rank- \(k\) SVD:

\[A \approx \hat{U}_k \hat{\Sigma}_k \hat{V}_k^T, \qquad \hat{U}_k \in \mathbb{R}^{m \times k},\ \hat{\Sigma}_k \in \mathbb{R}^{k \times k},\ \hat{V}_k \in \mathbb{R}^{n \times k}\]

minimizing \(\|A - \hat{U}_k \hat{\Sigma}_k \hat{V}_k^T\|_F\) subject to rank \(\leq k\).

The Eckart-Young theorem guarantees that the exact rank- \(k\) truncation (using the \(k\) largest singular values) is the best rank- \(k\) approximation. The randomized SVD approximates this optimum cheaply when \(\sigma_{k+1} \ll \sigma_k\) (fast spectral decay).

Reference: Halko, Martinsson, Tropp, "Finding Structure with Randomness" (SIAM Review 2011).

Key Insight: Randomized Range Finding

The column space of \(A\) is approximated by a small random sketch. If \(\Omega \in \mathbb{R}^{n \times \ell}\) is a Gaussian random matrix ( \(\ell = k + p\), \(p\) = oversampling), then:

\[Y = A\Omega \in \mathbb{R}^{m \times \ell}\]

has range \(\approx \operatorname{range}(A)\) with high probability. A QR factorization \(Y = Q\hat{R}\) gives an orthonormal basis \(Q\) for this approximate range.

The random sketch mixes right singular vectors with coefficients proportional to \(\sigma_i\), so large singular values dominate and the range of \(Y\) concentrates near the top- \(k\) singular subspace with high probability.

Error bound (Halko-Martinsson-Tropp): for Gaussian \(\Omega\) with \(\ell = k + p\), \(p \geq 2\):

\[\mathbb{E}\|A - QQ^TA\|_F \leq \left(1 + \frac{k}{p-1}\right)^{1/2} \left(\sum_{j > k} \sigma_j^2\right)^{1/2}\]

The right-hand side equals the optimal tail error \(\left(\sum_{j>k}\sigma_j^2\right)^{1/2}\) times a factor close to 1 for \(p \geq k\). In practice \(p = 10\) suffices.

Algorithm

function svd_truncated(A, k, p, rng):
m, n <- A.rows, A.cols
l <- min(k + p, min(m, n)) // sketch size (capped at matrix rank bound)
// Step 1: Gaussian random sketch Omega in R^{n x l}
Omega[i,j] ~ N(0, 1) iid
// Step 2: Y = A * Omega in R^{m x l}
Y <- A * Omega // DGEMM: O(mnl)
// Step 3: Q = thin QR(Y) in R^{m x l}, orthonormal columns
[Q, ~] <- thin_qr(Y) // O(ml^2)
// Step 4: B = Q^T * A in R^{l x n}
B <- Q^T * A // DGEMM: O(mnl)
// Step 5: Full SVD of the small B in R^{l x n}
[U_B, Sigma, Vt] <- svd(B) // O(l^2 n) or O(ln^2)
// Step 6: U = Q * U_B[:,0:k], keep top-k
U <- Q * U_B[:,0:k] // DGEMM: O(mlk)
return (U[:,0:k], Sigma[0:k], Vt[0:k, :])

Total cost: \(O(mn\ell)\) for the two DGEMM calls + \(O(m\ell^2)\) for QR + \(O(\ell^2 n)\) for the small SVD.

For \(\ell = k + p \ll \min(m, n)\): dramatically cheaper than exact SVD at \(O(mn\min(m,n))\).

Step-by-Step Analysis

Step 1: Random Sketch

The sketch \(\Omega \sim \mathcal{N}(0,1)^{n \times \ell}\) acts as a "sampler" of the column space. Alternative random matrices with similar guarantees:

  • Subsampled Randomized Hadamard Transform (SRHT): \(\Omega = \sqrt{n/\ell}\, D H S^T\) where \(D\) is a diagonal \(\pm 1\) matrix, \(H\) is the Walsh-Hadamard transform, and \(S\) selects \(\ell\) random columns. Cost: \(O(n \log n)\) per column instead of \(O(n\ell)\) for Gaussian – better for very large \(n\).
  • Sparse random maps (CountSketch): \(\Omega\) has one \(\pm 1\) per column, all others zero. Cost: \(O(\text{nnz}(A))\) for the sketch, works for sparse \(A\).
  • Structured random maps: FJLT (Fast Johnson-Lindenstrauss) – \(O(n \log \ell)\) per column.

Step 2: Matrix-Vector Product \f$Y = A\Omega\f$

This is a DGEMM: \(m \times n\) times \(n \times \ell\) -> \(m \times \ell\). Cost: \(2mn\ell\) flops.

For very large \(A\) (doesn't fit in RAM): compute \(Y\) in row blocks, accumulating columns of \(Y\) from partial sums:

\[Y = \sum_{b=1}^{B} A_b \Omega_b, \qquad A_b \in \mathbb{R}^{m_b \times n},\ \Omega_b = \Omega[\text{rows of }b, :]\]

Each block \(A_b\) is loaded from disk once, \(\Omega_b\) fits in RAM – only one pass over \(A\) required.

Step 3: QR of \f$Y\f$

Thin QR of \(m \times \ell\) matrix: \(O(m\ell^2)\) flops. For \(\ell \ll m\): cheap relative to the DGEMM.

Power iteration (optional, for slowly decaying spectra): replace \(Y = A\Omega\) with \(Y = (AA^T)^q A\Omega\) for some \(q \geq 1\):

\[Y \leftarrow A\Omega, \quad \text{then repeat } q \text{ times:} \quad Y \leftarrow A(A^T Y)\]

Each power step re-orthogonalizes \(Y\) via QR and costs \(2 \times O(mn\ell)\) extra flops. The effective singular value decay is raised to \(\sigma_j^{2q+1}\), dramatically improving approximation quality for slowly decaying spectra. Use \(q = 1\)– \(3\) for images or text data.

Step 4: Projection \f$B = Q^TA\f$

Another DGEMM: \(\ell \times m\) times \(m \times n\) -> \(\ell \times n\). Cost: \(2mn\ell\) flops. Together with Step 2, these two DGEMMs dominate the total cost.

Step 5: Small SVD of \f$B\f$

\(B \in \mathbb{R}^{\ell \times n}\) with \(\ell = k + p \ll n\): standard Jacobi SVD or dgesdd. Cost: \(O(\ell^2 n)\) (thin case, \(\ell < n\)) or \(O(\ell n^2)\) (fat case) – small in either case.

Step 6: Lift to Full Space \f$U = Q\hat{U}_\ell\f$

DGEMM: \(m \times \ell\) times \(\ell \times k\) -> \(m \times k\). Cost: \(2m\ell k\).

Oversampling Cap: Why \f$\ell \leq \min(m,n)\f$

If \(\ell > \min(m, n)\): the sketch \(Y = A\Omega \in \mathbb{R}^{m \times \ell}\) has rank at most \(\min(m,n)\). The thin QR of \(Y\) produces at most \(m\) orthonormal columns. Attempting to use \(\ell > m\) columns from a rank- \(m\) QR causes out-of-bounds access.

Fix (implemented in our code): cap \(\ell \leftarrow \min(k + p, \min(m, n))\) before proceeding.

Accuracy vs Cost Trade-off

Parameter Effect on accuracy Effect on cost
Oversampling \(p\) \(+\) improves (error factor \(\to 1\) as \(p \to \infty\)) \(+\) increases (linearly in \(p\))
Power iterations \(q\) \(+\) dramatically improves for slow decay \(+\) increases ( \(2q\) extra DGEMM passes)
Sketch type (Gaussian vs SRHT) similar guarantees Gaussian: \(O(mn\ell)\); SRHT: \(O(mn\log\ell)\)

Rule of thumb (Halko et al.):

  • \(p = 5\)- \(10\) for well-separated spectra (image compression, PCA on clean data)
  • \(q = 1\)- \(3\) power iterations for slowly decaying spectra (recommendation systems, text data)

Performance Optimization

Current Implementation

Two DGEMM calls (Steps 2 and 4) plus QR and small SVD. The DGEMM calls dominate for large \(m, n\).

BLAS optimization (already in use via Backend::blas): cblas_dgemm routes to AVX2/AVX-512 microkernel. For \(m = n = 10^4\), \(\ell = 50\): DGEMM achieves ~400 GFLOP/s vs ~0.5 GFLOP/s naive.

Streaming Computation (Out-of-Core)

For \(A\) too large for RAM (common in data science):

// Streaming Y = A * Omega in row blocks
Y <- 0 (m x l)
for each row-block A_b of A from disk:
Y += A_b * Omega[rows of b, :] // one DGEMM per block
// Then proceed: QR(Y), B = Q^T * A (second streaming pass), etc.

Cost: 2 passes over \(A\) (once for \(Y\), once for \(B\)). Memory: \(O(m\ell + n\ell)\) for \(Q\) and \(\Omega\).

GPU Implementation

Both DGEMM calls in Steps 2 and 4 are ideal for GPU:

// On GPU: cuBLAS cublasDgemm for Steps 2 and 4
// Gaussian sketch: cuRAND for fast generation of Omega
// QR: cuSOLVER cusolverDnDgeqrf (blocked Householder on GPU)
// Small SVD: host (l << n, cheap)

For \(m = n = 10^4\), \(\ell = 100\): GPU DGEMM at 10 TFLOP/s -> Steps 2+4 in \({\sim}2\) ms total. Compare with exact SVD (dgesdd): \(O(mn^2) \sim 10^{12}\) flops -> 1000 s on CPU, 100 s on GPU.

Avoiding Step 4 via Single-View Algorithm

If \(A\) is available only in a single pass (streaming):

// Single-view randomized SVD (Tropp et al. 2017)
// Generate two sketches simultaneously:
Y = A * Omega // right sketch (m x l)
Z = Psi^T * A // left sketch (l x n), Psi in R^{m x l}
// Recover B = Q^T A from Y and Z without a second pass over A:
Q, R_Y <- QR(Y)
B <- solve (R_Y^T B = Z) for B // O(l^2 n)

This uses only one pass over \(A\), at the cost of slightly worse accuracy. Needed for sensor data streams or when re-reading \(A\) is expensive.

Structured Sketches for Sparse \f$A\f$

For sparse \(A\) with \(\text{nnz} \ll mn\): use a CountSketch (one non-zero per column of \(\Omega\)):

\[Y_{ij} \leftarrow Y_{h(j), :} \mathrel{+}= \eta_j \cdot A[:, j], \quad h(j) \in \{0,\ldots,\ell-1\}, \quad \eta_j \in \{+1,-1\}\]

Cost: \(O(\text{nnz}(A))\) – same as one SpMV. The sketch sacrifices some accuracy (bounded by \(O(1/\sqrt{\ell})\) error) for extreme speed.
