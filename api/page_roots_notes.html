Root Finding

Given a continuous function \(f : \mathbb{R} \to \mathbb{R}\), find \(x^*\) such that \(f(x^*) = 0\).

Implementation: include/analysis/roots.hpp, src/analysis/roots.cpp

Bisection

The Intermediate Value Theorem guarantees that if \(f(a) \cdot f(b) < 0\), there exists a root in \([a, b]\). Bisection exploits this by repeatedly halving the interval.

Algorithm:

given [a, b] with f(a)*f(b) < 0
repeat:
mid = (a + b) / 2
if f(mid) ~= 0 or (b - a)/2 < tol: return mid
if f(a)*f(mid) < 0: b = mid
else: a = mid

Convergence: Linear. After \(n\) iterations, the error is at most \((b - a) / 2^n\).

\[|x_n - x^*| \leq \frac{b - a}{2^n}\]

To achieve tolerance \(\varepsilon\): \(n \geq \log_2\!\left(\frac{b-a}{\varepsilon}\right)\) iterations.

Properties: Guaranteed to converge. No derivative required. Slow.

Newton-Raphson

Expand \(f\) around the current estimate \(x_k\) via Taylor:

\[f(x^*) = f(x_k) + f'(x_k)(x^* - x_k) + O((x^* - x_k)^2) = 0\]

Dropping higher-order terms and solving for \(x^*\):

\[x_{k+1} = x_k - \frac{f(x_k)}{f'(x_k)}\]

Convergence: Quadratic near the root. If \(e_k = x_k - x^*\):

\[|e_{k+1}| \approx \frac{|f''(x^*)|}{2|f'(x^*)|} |e_k|^2\]

In practice, the number of correct decimal digits roughly doubles each iteration.

Properties: Fast near the root. Requires \(f'\). Can diverge if initial guess is poor or \(f'(x_k) \approx 0\).

Secant Method

Newton without the derivative. Approximate \(f'(x_k)\) by a finite difference using the two most recent iterates:

\[x_{k+1} = x_k - f(x_k) \cdot \frac{x_k - x_{k-1}}{f(x_k) - f(x_{k-1})}\]

Convergence: Superlinear with order \(\phi \approx 1.618\) (golden ratio):

\[|e_{k+1}| \approx C |e_k|^\phi\]

Slower than Newton but does not require a derivative.

Properties: Needs two initial points. Can stagnate if \(f(x_k) \approx f(x_{k-1})\).

Brent's Method

Combines bisection (guaranteed), secant (fast), and inverse quadratic interpolation (faster) in a single algorithm. At each step, it attempts the faster interpolation methods; if the proposed step is outside the current bracket or too large, it falls back to bisection.

Inverse quadratic interpolation: Given three points \((a, f(a)), (b, f(b)), (c, f(c))\), fit a quadratic through the three points in the \(x\)-direction (treating \(x\) as a function of \(f\)) and evaluate at \(f = 0\).

Invariant: \([b, c]\) always brackets the root. \(b\) is always the current best estimate.

Convergence: Superlinear in practice, with guaranteed worst-case linear convergence (from bisection fallback).

Properties: This is the recommended method when a bracket is available. No derivative required. Used in most production software (Brent 1973, also the basis of scipy.optimize.brentq).

Parallel Structure

Scalar root finding is inherently sequential – each iterate depends on the previous one.

Where parallelism appears:

  • Multiple independent roots: if \(f\) has \(k\) roots on \([a, b]\), each can be bracketed and solved in parallel.
  • Multivariate Newton (not yet implemented): each Newton step requires solving \(J \Delta x = -f(x)\), where \(J\) is the Jacobian. Computing \(J\) (finite differences) is embarrassingly parallel across columns.
  • Parameter sweeps: solving \(f(x; \lambda) = 0\) for many values of parameter \(\lambda\) is trivially parallel.

Exercises

  1. Bisection requires \(n = \lceil \log_2((b-a)/\varepsilon) \rceil\) iterations for tolerance \(\varepsilon\). How many iterations are needed to find \(\sqrt{2}\) to 15 significant figures starting from \([1, 2]\)?
  2. Show that Newton's method applied to \(f(x) = x^2 - c\) gives the Babylonian square root algorithm \(x_{k+1} = \frac{1}{2}(x_k + c/x_k)\).
  3. Newton's method can cycle on \(f(x) = x^3 - 2x + 2\) starting from \(x_0 = 0\). Show why, and suggest a fix.
  4. (Parallel) You need to find roots of \(f(x; \lambda) = \lambda x^3 - x - 1 = 0\) for 1024 values of \(\lambda \in [0.5, 2.0]\). Describe an OpenMP strategy. What is the expected parallel speedup?
