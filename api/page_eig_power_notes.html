Power Iteration, Inverse Iteration, and Rayleigh Quotient Iteration

Background

All three methods find one eigenvalue/eigenvector pair at a time by repeatedly applying a matrix (or its inverse) to a vector. They trade off cost per iteration against convergence rate:

| Method | Cost/iteration | Convergence | Finds | |-----—|------------—|----------—|----—| | Power iteration | 1 matvec \(O(n^2)\) | Linear, rate \(|\lambda_1/\lambda_2|\) | Dominant eigenvalue | | Inverse iteration | 1 triangular solve \(O(n^2)\) | Linear, rate \(|(\lambda-\sigma)/(\lambda_2-\sigma)|\) | Eigenvalue nearest \(\sigma\) | | Rayleigh quotient | 1 LU factorization \(O(n^3)\) | Cubic | Eigenvalue nearest starting guess |

Part I – Power Iteration

Algorithm

Expand the starting vector in the eigenbasis of \(A\): \(v_0 = \sum_i \alpha_i u_i\). After \(k\) multiplications:

\[A^k v_0 = \lambda_1^k \!\left(\alpha_1 u_1 + \sum_{i \geq 2} \alpha_i \left(\frac{\lambda_i}{\lambda_1}\right)^k u_i\right)\]

The component along \(u_i\) ( \(i \geq 2\)) decays as \((\lambda_i/\lambda_1)^k\) -> converges to \(u_1\).

function power_iteration(A, tol, max_iter):
v <- ones(n) / ||ones(n)|| // random start also valid
for iter = 1 to max_iter:
w <- A v // matvec: dominant cost O(n^2)
lambda <- v^T w // Rayleigh quotient (v is unit)
v <- w / ||w|| // renormalise
if |lambda - lambda_prev| < tol: return lambda, v, converged

Flops per iteration: \(2n^2\) (matvec) + \(2n\) (Rayleigh quotient) + \(n\) (normalise).

Convergence Analysis

Let \(|\lambda_1| > |\lambda_2| \geq \cdots \geq |\lambda_n|\). The asymptotic convergence factor is:

\[r = \left|\frac{\lambda_2}{\lambda_1}\right|\]

The angle between \(v_k\) and \(u_1\) satisfies:

\[\tan \angle(v_k, u_1) \leq \left|\frac{\lambda_2}{\lambda_1}\right|^k \frac{\|\alpha_{2:n}\|}{|\alpha_1|}\]

If \(\lambda_1 \gg \lambda_2\) (e.g., Google PageRank with \(r \approx 0.85\)): fast convergence. If \(|\lambda_1| \approx |\lambda_2|\): very slow – use Lanczos instead.

Failure Modes

  1. ** \(\alpha_1 = 0\)**: initial vector orthogonal to \(u_1\) -> converges to \(\lambda_2\) instead. Floating-point roundoff usually introduces a component along \(u_1\), but use a random start to be safe.
  2. ** \(|\lambda_1| = |\lambda_2|\)**: complex conjugate pair or \(\pm\) pair – iterate oscillates. Use 2-vector subspace iteration.
  3. Slow convergence: if \(r = 0.99\), \(\log(10)/\log(1/0.99) \approx 229\) iterations per digit.

Deflation (Hotelling)

After finding \((\lambda_1, u_1)\), subtract to expose \(\lambda_2\):

\[A' = A - \lambda_1 u_1 u_1^T \quad \Rightarrow \quad \text{eigenvalues of } A': \lambda_2, \lambda_3, \ldots, \lambda_n, 0\]

Errors accumulate with each deflation – numerically inferior to Lanczos or QR iteration for finding many eigenvalues.

Part II – Inverse Iteration

Algorithm

Replace \(A\) with \((A - \sigma I)^{-1}\). Its eigenvalues are \(\{1/(\lambda_i - \sigma)\}\), so the dominant eigenvalue of \((A-\sigma I)^{-1}\) corresponds to the \(\lambda_i\) closest to \(\sigma\):

function inverse_iteration(A, sigma, tol, max_iter):
// Factor (A - sigma*I) once: O(n^3)
LU <- lu_factor(A - sigma*I)
v <- ones(n) / ||ones(n)||
for iter = 1 to max_iter:
w <- lu_solve(LU, v) // O(n^2) triangular solve
v <- w / ||w||
lambda <- v^T A v // Rayleigh quotient
if |lambda - lambda_prev| < tol: return lambda, v, converged

Key advantage: factorize once (O(n^3)), then solve cheaply (O(n^2)) at each iteration.

Convergence

The convergence factor at shift \(\sigma\) is:

\[r = \frac{|\lambda - \sigma|}{|\lambda_2 - \sigma|}\]

where \(\lambda\) is the closest eigenvalue and \(\lambda_2\) is the second closest. Choosing \(\sigma\) close to the target makes \(r \approx 0\): typically 3-5 iterations suffice for machine precision once \(\sigma\) is within 10% of \(\lambda\).

Stability paradox: when \(\sigma \approx \lambda\) exactly, \((A - \sigma I)\) is nearly singular – the linear system is ill-conditioned. Yet the solution direction is extremely well-determined (the error in the eigenvalue is exponentially small). The most ill-conditioned solve gives the most accurate eigenvector.

Use Cases

  1. Refine a rough eigenvalue estimate: one inverse iteration step from within 10% of \(\lambda\) gives machine-precision accuracy
  2. Interior eigenvalue: choose \(\sigma\) in the middle of the spectrum – direct power iteration cannot reach this
  3. Sturm sequence + bisection: bisect on \([a,b]\) to isolate \(\lambda\), then inverse-iterate at \(\sigma = (a+b)/2\)

Part III – Rayleigh Quotient Iteration (RQI)

Algorithm

Update the shift \(\sigma\) at every iteration using the Rayleigh quotient of the current vector \(v\):

\[\sigma_k = \frac{v_k^T A v_k}{v_k^T v_k} = v_k^T A v_k \quad (\text{since } \|v_k\| = 1)\]

Fresh factorization at each step is expensive but gives cubic convergence:

function rayleigh_iteration(A, x0, tol, max_iter):
v <- x0 / ||x0||
sigma <- v^T A v
for iter = 1 to max_iter:
LU <- lu_factor(A - sigma*I) // O(n^3) per iter!
if singular: return sigma, v // sigma is exact eigenvalue
w <- lu_solve(LU, v)
v <- w / ||w||
sigma <- v^T A v
res <- ||A v - sigma v||_2
if res < tol: return sigma, v, converged

Cost per iteration: \(O(n^3)\) – same as a direct solve. Practical only for small \(n\) or as a refinement step (3 iterations at most from a good starting guess).

Convergence: Cubic Rate

For symmetric \(A\), the Rayleigh quotient is a second-order approximation to the eigenvalue:

\[\sigma = \lambda + O(\|v - u\|^2)\]

(the error in \(\sigma\) is quadratic in the eigenvector error). The new eigenvector error is proportional to \(|\sigma - \lambda|\) -> quadratic in the old eigenvector error -> cubic overall:

\[|\sigma_{k+1} - \lambda| \leq C\, |\sigma_k - \lambda|^3\]

Consequence: if the current estimate has error \(10^{-4}\), the next has error \(\sim 10^{-12}\) – machine precision in one more step.

For non-symmetric \(A\), only quadratic convergence (the second-order property of the Rayleigh quotient holds for symmetric matrices only).

Practical Strategy: Combine Methods

Phase 1: Lanczos (k steps) -> Ritz values lambda~_1, ..., lambda~_k O(k*nnz) or O(kn^2)
Phase 2: One inverse iteration step at sigma = lambda~i O(n^3) x few steps

Or for dense \(A\):

Phase 1: power_iteration or inverse_iteration -> rough (lambda, v)
Phase 2: one more inverse_iteration at sigma = lambda -> machine precision

Performance Optimization

SIMD for Power Iteration Matvec

Power iteration's only per-iteration cost is the matvec \(w \leftarrow Av\). For row-major dense \(A\):

for (idx i = 0; i < n; ++i) {
__m256d acc = _mm256_setzero_pd();
for (idx j = 0; j < n; j += 4)
acc = _mm256_fmadd_pd(_mm256_load_pd(&A(i,j)),
_mm256_load_pd(&v[j]), acc);
w[i] = hsum(acc);
}

For sparse \(A\): use library SpMV (MKL mkl_dcsrmv, cuSPARSE cusparseDbsrmv).

Subspace Iteration (Block Power Method)

Instead of one vector, maintain \(V \in \mathbb{R}^{n \times k}\) and apply \(A\):

\[W \leftarrow AV \quad (\text{DGEMM: } O(n^2 k)), \qquad [Q, R] \leftarrow \text{thin QR}(W), \qquad V \leftarrow Q\]

Simultaneously finds the \(k\) dominant eigenpairs. Convergence factors:

\[r_i = \left|\frac{\lambda_{k+1}}{\lambda_i}\right|, \quad i = 1, \ldots, k\]

The blocked version amortizes matvec overhead (DGEMM \(\gg\) GEMV in practice) and is the precursor to the Lanczos algorithm.

Blocked Inverse Iteration for Multiple Eigenvectors

For tridiagonal \(T\) (from Lanczos), inverse-iterate all desired eigenvectors simultaneously:

\[\text{Solve } (T - \sigma I)\,X = B \quad \text{(all RHS simultaneously)}\]

Replace \(k\) DTRSV calls with one DTRSM call (BLAS-3) -> much higher arithmetic intensity.

Shift Selection via Gershgorin Discs

Eigenvalues of \(A\) lie in the union of Gershgorin discs:

\[\sigma(A) \subseteq \bigcup_{i=1}^n \bigl\{ z \in \mathbb{C} : |z - A_{ii}| \leq R_i \bigr\}, \qquad R_i = \sum_{j \neq i} |A_{ij}|\]

Use Gershgorin to get rough bounds for initial shift selection in inverse iteration. For SPD matrices, all discs lie on the positive real axis: \([A_{ii} - R_i, A_{ii} + R_i] \cap \mathbb{R}_{>0}\).

Sturm sequence bisection for SPD tridiagonal \(T\): the number of eigenvalues of \(T\) less than \(\mu\) equals the number of sign changes in the sequence \(\{d_k\}\) defined by:

\[d_0 = T_{00} - \mu, \qquad d_k = (T_{kk} - \mu) - \frac{T_{k-1,k}^2}{d_{k-1}}\]

Bisect on \([a, b]\) to isolate each eigenvalue to within \(\varepsilon\) in \(O(n \log(1/\varepsilon))\) flops.
