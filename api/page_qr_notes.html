QR Factorization via Householder Reflections

Overview

Factors \(A \in \mathbb{R}^{m \times n}\) ( \(m \geq n\)) as \(A = QR\) where \(Q \in \mathbb{R}^{m \times m}\) is orthogonal and \(R \in \mathbb{R}^{m \times n}\) is upper triangular. The economy (thin) form uses \(\hat{Q} \in \mathbb{R}^{m \times n}\) and \(\hat{R} \in \mathbb{R}^{n \times n}\).

When to use:

  • Least-squares: \(\min_x \|Ax - b\|_2\) with \(m > n\) (overdetermined)
  • Eigenvalue algorithms: QR iteration for non-symmetric \(A\)
  • Krylov orthogonalization: Gram-Schmidt in GMRES and Lanczos

Householder Reflections

A Householder reflector is the rank-1 symmetric orthogonal matrix

\[H = I - \frac{2vv^T}{v^Tv}, \qquad H^T = H, \quad H^2 = I, \quad \det(H) = -1\]

For a vector \(x \in \mathbb{R}^m\), choose \(v\) so that \(H\) maps \(x\) to a scalar multiple of \(e_1\):

\[Hx = -\operatorname{sign}(x_1)\|x\|_2\, e_1\]

Construction:

\[\sigma \leftarrow \operatorname{sign}(x_1)\|x\|_2, \qquad v \leftarrow x + \sigma e_1, \qquad v \leftarrow v/\|v\|\]

Sign convention: choose the same sign as \(x_1\). This avoids catastrophic cancellation: if \(x_1 > 0\), then \(v_1 = x_1 + \|x\| > 0\) is a sum of positives. The alternative \(v_1 = x_1 - \|x\|\) would subtract nearly equal numbers when \(x_1 \approx \|x\|\).

Algorithm

Apply \(r = \min(m-1, n)\) reflections. Step \(k\) zeros all entries of column \(k\) below the diagonal.

function QR(A):
m, n <- A.rows, A.cols
r <- min(m-1, n)
R <- copy of A
vs <- list of r Householder vectors
for k = 0 to r-1:
x <- R[k:m, k] // column stub (length m-k)
sigma <- sign(x[0]) * ||x||
v <- x; v[0] += sigma // unnormalised Householder vector
v /= ||v||; vs[k] <- v
// Apply H_k = I - 2vv^T to trailing submatrix R[k:m, k:n]
// For each column j: R[k:m,j] -= 2v * (v^T R[k:m,j])
for j = k to n-1:
tau <- 2 * (v^T R[k:m, j])
R[k:m, j] -= tau * v
// Build Q by accumulating reflectors in reverse order
Q <- I_m
for k = r-1 downto 0:
v <- vs[k]
for j = k to m-1: // columns j < k are already correct (= e_j)
tau <- 2 * (v^T Q[k:m, j])
Q[k:m, j] -= tau * v
// Zero sub-diagonal of R (kill floating-point noise)
R[i,j] <- 0 for all i > j
return (Q, R)

Complexity:

  • Factorization: \(2mn^2 - \frac{2}{3}n^3\) flops
  • Building \(Q\): \(4m^2n - 2mn^2 + \frac{2}{3}n^3\) flops (dominant for \(m \gg n\))

If only \(Q^Tb\) is needed (least-squares), **never build \(Q\) explicitly** – apply each \(H_k\) to \(b\) in \(O(mn)\) total.

Least-Squares Solve

\(\min_x \|Ax - b\|_2\) has normal equations \(A^TAx = A^Tb\), but solving those squares the condition number. The QR approach is numerically preferable:

\[\|Ax - b\|_2^2 = \|QRx - b\|_2^2 = \|Rx - Q^Tb\|_2^2 = \|\hat{R}x - \tilde{c}\|_2^2 + \|\bar{c}\|_2^2\]

where \(c = Q^Tb\), \(\tilde{c} = c[0:n]\) (matched components), \(\bar{c} = c[n:m]\) (residual).

function QR_solve(Q, R, b):
c <- Q^T b // O(mn): matrix-vector with Q transposed
x <- back-solve R[0:n, 0:n] x = c[0:n] // O(n^2)
return x

Stability

QR via Householder is backward stable independent of \(\kappa(A)\), unlike the normal equations \(A^TAx = A^Tb\) which square the condition number. For \(\kappa(A) > 10^8\) in double precision, normal equations lose all significant digits while QR still gives correct residuals.

Alternatives: Givens vs MGS vs Householder

Method Flops Stability Use case
Householder \(2mn^2 - \frac{2}{3}n^3\) Backward stable Dense QR, standard
Modified Gram-Schmidt (MGS) \(2mn^2\) Conditionally stable Krylov (Arnoldi), thin \(Q\)
Classical Gram-Schmidt (CGS) \(2mn^2\) Unstable Never use alone
Givens rotations \(\sim 3mn^2\) Backward stable Banded/sparse, rank-1 updates

Givens is preferred when \(A\) has structured sparsity: each rotation \(G(i,j,\theta)\) modifies only rows \(i\) and \(j\), zeroing one element without disturbing the rest. For dense \(A\), Householder is preferred – one reflector zeros an entire column stub.

Performance Optimization

Current Implementation

The apply-reflector inner loop processes columns one at a time:

for j = k to n-1:
tau <- 2 * (v^T R[k:m, j]) // dot product: length m-k
R[k:m, j] -= tau * v // AXPY: length m-k

With row-major storage, R[k:m, j] (a column) is strided – every access is a cache miss for large \(m\). This is the dominant cost for tall matrices.

Blocked Householder: WY Representation

Accumulate a block of \(n_b\) Householder vectors into a compact WY form:

\[H_1 H_2 \cdots H_{n_b} = I - WY^T, \qquad W, Y \in \mathbb{R}^{m \times n_b}\]

Apply the entire block to the trailing submatrix in one DGEMM:

\[A \leftarrow A - W(Y^T A) \qquad \text{(2 DGEMM calls)}\]

This is a BLAS-3 operation with arithmetic intensity \(O(n_b)\). LAPACK uses this in dgeqrf with \(n_b \approx 64\).

Building the WY representation: \(O(m \times n_b^2)\) – the same as the unblocked panel factorization, negligible relative to the trailing DGEMM.

SIMD for the AXPY Kernel

The per-column AXPY R[k:m,j] -= tau * v for length \(\ell = m - k\):

__m256d vtau = _mm256_broadcast_sd(&tau);
for (idx i = 0; i < len; i += 4) {
__m256d vv = _mm256_load_pd(v + i);
__m256d vr = _mm256_load_pd(R_col + i);
vr = _mm256_fnmadd_pd(vtau, vv, vr); // r -= tau * v
_mm256_store_pd(R_col + i, vr);
}

For strided column access (row-major \(A\)): transpose the panel into a column-major buffer before applying reflectors, then transpose back. This converts \(O(mn/4)\) gather operations into \(O(m)\) sequential reads.

Avoiding Explicit Q

In practice \(Q\) is almost never needed. Store the compact reflectors and apply \(Q^T\) on-demand in \(O(mn)\):

// Apply Q^T to vector b without forming Q
for k = 0 to r-1:
tau <- 2 * (vs[k]^T b[k:m])
b[k:m] -= tau * vs[k]

This halves memory usage (no \(m \times m\) matrix) and streams through \(A\) once.

Tall-Skinny QR (TSQR) for Distributed Systems

For \(m \gg n\) (e.g., \(m = 10^6\), \(n = 100\)), use a communication-avoiding tree reduction:

  1. Partition \(A\) into \(p\) row blocks: \(A = [A_1;\, A_2;\, \ldots;\, A_p]\)
  2. Factor each \(A_i \to Q_i R_i\) in parallel (each block fits in L2)
  3. Stack the \(R\) blocks: \([R_1;\, R_2;\, \ldots;\, R_p] \xrightarrow{\text{QR}} R\) (small \(p \times n\) problem)

TSQR achieves the same backward stability as standard Householder with communication volume \(O(n^2 \log p)\) vs \(O(mn)\) for column-panel QR. It is the basis for Communication-Avoiding QR (CAQR) in ScaLAPACK and SLATE.
