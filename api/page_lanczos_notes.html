Lanczos Algorithm with Full Reorthogonalization

Problem Statement

Given a symmetric matrix \(A \in \mathbb{R}^{n \times n}\) (or a symmetric linear operator defined by a matvec), find the \(k\) dominant or extreme eigenpairs efficiently when \(k \ll n\). Lanczos builds a \(k\)-dimensional Krylov subspace containing near-optimal approximations to the extreme eigenvalues.

Applications: graph Laplacian spectral analysis, quantum chemistry (CI), structural vibration modes, PCA via SVD, spectral graph clustering.

The Lanczos Recurrence

Starting from a unit vector \(v_1\), the \(k\)-step Lanczos process builds an orthonormal basis \(V_k = [v_1, \ldots, v_k]\) for the Krylov subspace \(\mathcal{K}_k(A, v_1)\) satisfying:

\[A V_k = V_k T_k + \beta_k v_{k+1} e_k^T\]

where \(T_k\) is the symmetric tridiagonal matrix:

\[T_k = \begin{pmatrix} \alpha_1 & \beta_1 & & \\ \beta_1 & \alpha_2 & \beta_2 & \\ & \ddots & \ddots & \ddots \\ & & \beta_{k-1} & \alpha_k \end{pmatrix}\]

The three-term recurrence generating each new basis vector:

\[w = A v_j, \qquad \alpha_j = v_j^T w, \qquad w \leftarrow w - \alpha_j v_j - \beta_{j-1} v_{j-1}, \qquad \beta_j = \|w\|, \qquad v_{j+1} = w/\beta_j\]

This is the Lanczos relation. In exact arithmetic, the three-term recurrence alone maintains orthogonality. In floating-point, rounding errors cause loss of orthogonality -> ghost eigenvalues -> full reorthogonalization is required.

Algorithm: Lanczos with Full Reorthogonalization (MGS)

function lanczos(matvec, n, k, tol, max_steps):
max_steps <- min(3k, n) if not specified
V[n x max_steps] <- 0
alpha[max_steps] <- 0
beta[max_steps] <- 0
V[:,0] <- e_1 // deterministic start
for j = 0 to max_steps-1:
vj <- V[:,j]
w <- matvec(vj) // dominant cost: O(n^2) dense, O(nnz) sparse
// Three-term recurrence
alpha[j] <- vj^T w
w <- w - alpha[j]*vj
if j > 0: w <- w - beta[j-1] * V[:,j-1]
// Full reorthogonalization: project w onto orthogonal complement of span{v_0,...,v_j}
// \f$w \leftarrow w - \sum_{l=0}^{j} (v_l^T w)\, v_l\f$
for l = 0 to j:
proj <- V[:,l]^T w
w <- w - proj * V[:,l]
beta[j] <- ||w||
if beta[j] < 1e-12: break // invariant subspace
if j+1 < max_steps: V[:,j+1] <- w / beta[j]
// Build m x m tridiagonal T (m = steps taken)
T[j,j] <- alpha[j]; T[j,j+1] = T[j+1,j] <- beta[j]
// Eigendecompose T: O(m^3) -- negligible for m << n
(lambda_tilde, S) <- eig_sym(T)
// Ritz vectors: V_m * (columns of S), cost O(mn)
U_tilde[r,i] = sum_j S[j,i] * V[r,j]
// Convergence check: ||A u_i - lambda_i u_i||_2 for each Ritz pair
return (lambda_tilde, U_tilde, steps, converged)

Total cost: \(O(k \cdot \text{nnz})\) (matvecs) \(+ O(k^2 n)\) (reorthogonalization) \(+ O(k^3)\) (inner eigensolver, negligible).

The Projection Principle

The tridiagonal \(T_k\) is the Galerkin projection of \(A\) onto \(\mathcal{K}_k\):

\[T_k = V_k^T A V_k \in \mathbb{R}^{k \times k}\]

Its eigenvalues – the Ritz values \(\theta_1, \ldots, \theta_k\) – are the best polynomial approximations to \(A\)'s eigenvalues within \(\mathcal{K}_k\). Specifically, \(\theta_i\) minimizes \(|p(\lambda_i)|^2\) over all degree- \(k\) polynomials \(p\) with \(p(0) = 1\) in the subspace sense.

The Ritz vectors \(\tilde{u}_i = V_k s_i\) (columns of \(V_k S\)) satisfy:

\[\|A\tilde{u}_i - \theta_i \tilde{u}_i\|_2 = \beta_k |S_{k,i}| \qquad \text{(cheap convergence estimate!)}\]

Convergence order: extreme eigenvalues ( \(\lambda_1\), \(\lambda_n\)) converge first; interior eigenvalues last.

Convergence

Convergence is governed by the spectral gap \(\delta = (\lambda_1 - \lambda_2)/(\lambda_1 - \lambda_n)\):

  • Large gap ( \(\delta \gg 0\)): leading Ritz value converges in \(O(1/\sqrt{\delta})\) steps
  • Clustered extreme eigenvalues: slow convergence, need large \(k\)

The \(\beta_k |S_{k,i}|\) residual estimate from the algorithm provides an inexpensive stopping criterion – no need to compute the full Ritz vector to check convergence.

Ghost Eigenvalues and Reorthogonalization

In floating-point, the three-term recurrence loses orthogonality after many steps. Once a Ritz value converges to \(\lambda_j\), the Lanczos vector drifts back into the \(u_j\) eigenspace, causing \(\lambda_j\) to appear again as a spurious ghost eigenvalue.

Full reorthogonalization (FRO) (our implementation):

  • Project \(w\) against all \(v_0, \ldots, v_j\) at each step via \(w \leftarrow w - \sum_{l=0}^{j} (v_l^T w)\, v_l\)
  • Cost: \(O(jn)\) per step, \(O(k^2 n)\) total
  • Guarantees orthogonality to machine precision
  • Memory: must store all \(k\) vectors simultaneously

Performance Optimization

BLAS-2 Reorthogonalization

MGS reorthogonalization at step \(j\) performs \(j+1\) BLAS-1 DOT + AXPY pairs. Replace with BLAS-2 DGEMV:

\[h = V[:,0:j+1]^T w \quad \to \quad \texttt{dgemv}(V[:,0:j+1]^T, w) \qquad O(jn), \text{ one call}\]

\[w \leftarrow w - V[:,0:j+1]\, h \quad \to \quad \texttt{dgemv}(V[:,0:j+1], h) \qquad O(jn), \text{ one call}\]

Replaces \(j\) DDOT \(+ j\) DAXPY with 2 DGEMV calls: \(V[:,0:j+1]\) is loaded once from cache for both operations.

Block Lanczos (BLAS-3)

Replace scalar Lanczos vectors with a block of \(b\) vectors. The recurrence becomes:

\[W = A\, V[:,j:j+b] \qquad \to \qquad \text{DGEMM: } O(n^2 b)\]

\[H = V[:,0:j]^T W \qquad \to \qquad \text{DGEMM: } O(jnb)\]

\[W \leftarrow W - V[:,0:j]\, H \qquad \to \qquad \text{DGEMM: } O(jnb)\]

\[[Q, R] = \text{thin QR}(W) \qquad \to \qquad \text{DGEQRF: } O(nb^2)\]

The tridiagonal \(T_k\) becomes a block tridiagonal with \(b \times b\) diagonal blocks. Advantages: BLAS-3 for all major operations -> near-peak GFLOP/s; naturally resolves \(b\)-fold degenerate eigenvalues.

Sparse Matrix Formats for SpMV

Format Memory Access Best for
CSR \(2\,\text{nnz} + n + 1\) Irregular column General sparse
ELLPACK \(\text{nnz}_{\text{padded}}\) Uniform row width Regular sparsity, GPU
BCSR (block) \(\sim \text{nnz}\) Better locality Block FEM/FD
Stencil (no storage) \(O(1)\) Streaming Structured grids

For structured grids (finite differences), implement the matvec as a stencil loop with no matrix storage at all – perfectly streaming, SIMD-friendly, and memory-optimal.

Distributed Lanczos (MPI)

Distribute rows of \(A\) and \(V\) across \(p\) processes. Each process holds rows \([i_{\text{start}}, i_{\text{end}})\):

// Local SpMV + Allreduce boundary exchange
w_local <- A_local * vj_local
MPI_Allreduce(w_local, w, SUM) // one global reduction per matvec
// Local dot product + Allreduce
alpha_local <- vj_local^T w_local
MPI_Allreduce(&alpha_local, &alpha, SUM)

Communication per step: 2 Allreduce operations = \(O(\log p)\) latency. Total for \(k\) steps: \(O(k \log p)\) latency synchronizations. The inner eigensolver on \(T\) is replicated on all processes (small \(k \times k\), \(O(k^3)\)).

For large \(k\) (k > 1000), the reorthogonalization step V[:,0:j]^T w requires a distributed PDGEMV in ScaLAPACK or a custom blocked DGEMV across the partitioned \(V\) matrix.
